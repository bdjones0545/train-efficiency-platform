Build Unified CEO Heartbeat + Agent Operating System

Objective:
TrainEfficiency currently has many working agents and background jobs, but they run as separate systems. Build a unified CEO Heartbeat layer that coordinates all agents, workflows, approvals, outcomes, and execution logs from one operating center.

⸻

Part 1 — Create CEO Heartbeat Service

Create:

server/services/ceo-heartbeat-service.ts

Purpose:
Run one central orchestration cycle that reviews the entire business state and delegates work across all agent systems.

The CEO Heartbeat should coordinate:

* Executive Agent
* Retention Agent
* Revenue Agent
* Gmail Agent
* Domain Outreach Service
* AI Comms Center
* Auto-Execution Engine
* Outcome Intelligence Service
* Scheduling Agent
* Daily Operations Engine
* TrainChat / Workout Builder signals
* Workflow Orchestrator

Run cadence:

* Every 30 minutes
* Manual trigger from admin UI

⸻

⸻

Part 3 — Replace Disconnected Cron Logic With Coordinated Heartbeat

Do not delete existing jobs immediately.

Instead:

1. Keep existing crons.
2. Add execution locks and idempotency.
3. Route their outputs into the CEO Heartbeat timeline.
4. Gradually let CEO Heartbeat become the central coordinator.

Add lock table:

agent_execution_locks

Fields:

* id
* org_id
* job_name
* lock_key
* acquired_at
* expires_at
* released_at
* status

Every recurring job must acquire a lock before running.

⸻

Part 4 — Add Idempotency Keys

Add idempotency protection for external actions.

For every email send, booking action, workflow execution, and outreach generation:

* generate idempotency_key
* check if already executed
* skip duplicate execution
* log skipped_duplicate in timeline

Suggested key:

orgId:agentName:actionType:entityId:messageType:dateBucket

Acceptance:
Server restart or duplicate cron cannot send the same email twice.

⸻

Part 5 — Outcome Bridge Completion

Complete automatic outcome linking.

When Gmail reply detected:

* update agent_communication_outcomes
* set outcome_status = replied
* set replied_at
* write timeline event

When booking is created from lead:

* update related outcome row
* set outcome_status = booked_session
* set booked_session_at
* attach revenue_cents
* write timeline event

When deal stage changes:

* update outcome row:
    * proposal_requested
    * proposal_sent
    * proposal_accepted
    * contract_signed
    * converted
    * lost

When employment applicant stage changes:

* update outcome row:
    * interview_requested
    * interviewed
    * offer_sent
    * hired
    * rejected

Acceptance Criteria:

* One CEO Heartbeat coordinates all major agent systems.
* Every agent action appears in one operating timeline.
* No duplicate emails can be sent from cron overlap.
* Replies/bookings/deals/hiring outcomes update communication outcomes automatically.
* Admins can see top priorities from all agents in one dashboard.
* Emergency pause stops future execution.
* Admin actions are audited.
* Existing agents still work.
