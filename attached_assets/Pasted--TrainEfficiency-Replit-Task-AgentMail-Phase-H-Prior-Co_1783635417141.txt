# Goal

Measure whether relationship-aware drafts perform better than baseline drafts.

The system should track:

* Was prior contact context used?
* What prior context was used?
* Did drafts with prior context get more replies?
* Did they lead to evaluations, conversions, or onboarding outcomes?
* Which communication domains benefit most from prior contact context?

# Scope

## 1. Persist Prior Contact Metadata

Whenever a draft generation path calls the prior contact context service and the returned context is non-empty, write metadata into `gmail_agent_actions.result`.

Add fields such as:

```ts
{
  priorContactUsed: true,
  priorContactSummary: {
    sentCount: number,
    replyCount: number,
    lastSentAt?: string,
    lastOutcome?: string,
    lastSubject?: string,
    lastDomain?: string,
    evaluationScheduled: boolean,
    converted: boolean,
    firstSessionScheduled: boolean,
    programAssigned: boolean,
    paymentRecovered: boolean,
    recommendedCTA?: string,
    recommendedTone?: string
  }
}
```

If no prior context exists:

```ts
{
  priorContactUsed: false
}
```

Rules:

* Do not store full previous email bodies.
* Do not store sensitive private notes.
* Store only summary metadata.
* Preserve existing `result` JSONB fields.
* Do not overwrite `fullBody`, `subject`, workflow metadata, or saved edits.
* Fail open if metadata write fails.

---

## 2. Wire Persistence Into All Phase G Paths

Update every generation path that uses prior contact context:

* `intelligent-lead-intake-service.ts`

If a path only knows context after insert, patch the result JSONB after insert.

## 4. Add Prior Context Comparison Endpoint

Add or update endpoint:

```txt
GET /api/admin/agentmail-learning/prior-context-analytics
```

or use existing endpoint from Phase G.

Return:

```ts
{
  totals: {
    draftsWithPriorContext,
    draftsWithoutPriorContext,
    replyRateWithContext,
    replyRateWithoutContext,
    conversionRateWithContext,
    conversionRateWithoutContext,
    evaluationRateWithContext,
    evaluationRateWithoutContext
  },
  byDomain: [
    {
      domain,
      withContextCount,
      withoutContextCount,
      replyRateWithContext,
      replyRateWithoutContext,
      conversionRateWithContext,
      conversionRateWithoutContext,
      evaluationRateWithContext,
      evaluationRateWithoutContext,
      dataConfidence: "none" | "low" | "medium" | "high",
      interpretation: string
    }
  ],
  recentExamples: [
    {
      actionId,
      domain,
      recipientEmail,
      priorContactSummary,
      outcomes: []
    }
  ]
}
```

Confidence levels:

* none: no prior-context drafts
* low: 1–4 prior-context drafts
* medium: 5–19 prior-context drafts
* high: 20+
