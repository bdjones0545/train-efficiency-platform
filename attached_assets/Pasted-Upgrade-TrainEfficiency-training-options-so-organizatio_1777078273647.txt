Upgrade TrainEfficiency training options so organizations can define session categories, client pricing, and coach payout logic separately.

Current issue:
Training options only support basic session type, duration, and client price. This does not handle real business cases like:
- free intro sessions
- floor hours
- meetings
- membership/package redemptions
- internal paid coach time
- $0 client price but paid coach compensation

Goal:
Separate revenue logic from coach payout logic.

Do not treat all $0 sessions as the same. A $0 intro, $0 floor hour, and $0 membership redemption are different business events.

========================
PHASE 2 — PAYOUT CALCULATION LOGIC
========================

Create or update a centralized function:

calculateCoachPayoutForBooking(booking)

Inputs:
- training option
- booking duration
- client price
- assigned coach payout defaults
- training option payout settings

Rules:
1. If payoutType = none:
   coach payout = 0

2. If payoutType = fixed:
   coach payout = payoutValueCents

3. If payoutType = hourly:
   coach payout = payoutValueCents × booking duration hours

4. If payoutType = percentage:
   coach payout = eligible revenue × payoutPercent

Eligible revenue:
- if countsTowardRevenue = true → use booking price
- if revenueRecognition = at_purchase → use redemption value if available
- if priceCents = 0 and no redemption value → percentage payout should be 0 unless fixed override exists

Important:
- Floor hours with hourly payout should pay coach even when client price is $0.
- Meetings may pay or not pay depending on payoutType.
- Intro sessions may pay fixed coach amount even with $0 client price.
- Package redemptions should be able to pay coach even if booking price is $0.

Return:
- coachPayoutCents
- payoutSource:
  - booking_price
  - fixed
  - hourly
  - package_redemption
  - none
- payoutExplanation
