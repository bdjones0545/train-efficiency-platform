## 2. Outcome Table

If no existing table exists, create:

```sql
agent_communication_outcomes (
  id varchar primary key default gen_random_uuid(),
  org_id varchar not null,
  action_id varchar not null,
  recipient_email text,
  communication_domain varchar,
  outcome_type varchar not null,
  outcome_value numeric,
  related_entity_type varchar,
  related_entity_id varchar,
  detected_at timestamp default now(),
  source varchar not null,
  confidence numeric default 1.0,
  metadata jsonb default '{}',
  created_at timestamp default now()
)
```

Add unique protection:

```sql
unique(org_id, action_id, outcome_type, related_entity_type, related_entity_id)
```

## 3. Correlation Rules

Implement deterministic rules.

Examples:

### Reply Received

If an inbound reply exists after the sent email:

```txt
outcome_type = reply_received
```

### Evaluation Scheduled

If linked lead has booking/evaluation status after email sent:

```txt
outcome_type = evaluation_scheduled
```

### Lead Converted

If linked lead becomes enrolled after email sent:

```txt
outcome_type = lead_converted
```

### First Session Scheduled

If onboarding checklist firstSessionScheduled becomes true after email sent:

```txt
outcome_type = first_session_scheduled
```

### Program Assigned

If programAssigned becomes true after email sent:

```txt
outcome_type = program_assigned
```

### Payment Recovered

If payment recovery email is followed by paid invoice / wallet balance resolution / payment setup:

```txt
outcome_type = payment_recovered
```

Only implement this if real billing data exists.

Otherwise return not configured.

## 4. Correlation Job / Endpoint

Add endpoint:

```txt
POST /api/admin/agentmail-outcomes/recompute
```

For admin-triggered recompute.

Also add read endpoint:

```txt
GET /api/admin/agentmail-outcomes/summary?range=7d|30d|90d&domain=
```

Return:

```ts
{

    sentMessages,
    replies,
    evaluationsScheduled,

    firstSessionsScheduled,
    programsAssigned,

```
