TrainEfficiency Sprint 1 — Org Isolation & Security Hardening

Objective

Eliminate all cross-organization data leakage risk and establish a single authoritative organization resolution path across the entire platform.

This sprint should be completed before any new agent, heartbeat, workflow, or automation work.

Success Criteria:

* No route resolves orgId in more than one way.
* No route falls back to “” when org resolution fails.
* Every org-scoped route either resolves an org or returns 403.
* No authenticated user can access another organization’s data.
* No public endpoint can be used to enumerate organization data.

⸻

Phase 1 — Create Single Source of Truth

New Utility

Create:

server/lib/resolve-org-id.ts

Export:

resolveOrgIdOrThrow(req)

Behavior:

1. Attempt resolveOrgSession(req)
2. Verify organization exists
3. Verify authenticated user belongs to organization
4. Return orgId
5. If org cannot be resolved:

throw new Error(“ORG_RESOLUTION_FAILED”)

Route middleware converts to:

403 Forbidden

Never return:

null
undefined
“”

Forbidden patterns:

req.user?.organizationId || “”
req.session?.organizationId || “”
req.query.organizationId || “”
req.body.organizationId || “”

outside of the resolver.

⸻

Phase 4 — Fix Critical Leakage Routes

Route A

GET /api/services

Current risk:

Returns all services when organizationId omitted.

Required:

const orgId = await resolveOrgIdOrThrow(req)

Validation:
