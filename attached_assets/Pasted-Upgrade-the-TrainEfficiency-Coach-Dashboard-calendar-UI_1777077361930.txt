Upgrade the TrainEfficiency Coach Dashboard calendar UI so coaches can schedule sessions at precise times like 10:30 AM without manually typing the time.

Do not change backend scheduling rules unless needed to pass selected startTime/endTime. This is primarily a calendar UI/UX improvement.

========================
PHASE 1 — ADD 15/30 MINUTE SLOT SELECTION
========================

Current issue:
The calendar only exposes large hour windows, so tapping 10 AM makes it hard to schedule 10:30 AM.

Update the daily calendar grid to support selectable time increments.

Preferred:
- 30-minute visible slots by default
- Optional 15-minute precision if service duration or settings require it

Examples:
- 10:00
- 10:30
- 11:00
- 11:30

When a coach taps 10:30:
- open Add Session modal
- prefill start time = 10:30 AM
- prefill end time based on selected service duration
- preserve selected coach/date

========================
PHASE 2 — TAP + DRAG BEHAVIOR
========================

Support mobile-friendly selection:

Tap:
- single tap on a slot selects that start time

Long press or drag:
- optional stretch selection to choose custom duration
- example: drag from 10:30 to 11:30 creates 60-minute block

If drag is too much for now, skip it and implement reliable tap selection first.

========================
PHASE 3 — IMPROVE EMPTY SLOT VISUALS
========================

Show subtle clickable half-hour rows.

Current calendar feels like large empty blocks.

Improve:
- show faint divider lines at 30-minute intervals
- make available ranges lightly tinted
- show a small plus icon on hover/tap-ready slots
- on mobile, show plus icon centered in available open blocks

Important:
Do not make the UI cluttered.
The coach should immediately know:
“This is an available time I can tap.”

========================
PHASE 4 — QUICK TIME PICKER AFTER TAP
========================

If the coach taps an hour block instead of a precise slot, show quick options:

Example after tapping 10 AM:
- 10:00
- 10:15
- 10:30
- 10:45

This solves precision without requiring a dense grid.

Mobile UX:
- bottom sheet with time chips
- title: “Choose start time”
- then open Add Session modal with that time

========================

Current issue:
The agent launcher may overlap lower calendar slots.

Adjust:

- avoid covering active calendar slots

- ensure plus/tap areas near bottom remain usable
