Build the AgentMail Inbound Routing Layer for TrainEfficiency.

Goal:
When emails come into AgentMail inboxes like revenue@, hiring@, scheduling@, support@, operations@, or ceo@, TrainEfficiency should classify the message, attach it to the correct organization, create an actionable item, and route it to the correct agent workflow.

Before coding:
Inspect the existing AgentMail integration, agentmail-service.ts, agentmail-routes.ts, agent mail tables, agent workflows, Attention Inbox, CEO Heartbeat, Revenue Agent, Scheduling Agent, Hiring/Employment Agent, Support/Client Success Agent, and organization/auth patterns.

Core requirements:

1. Webhook ingestion
Update the existing AgentMail webhook handler so inbound email events are captured and normalized.

Store:
- organizationId
- inbox
- fromEmail
- fromName
- toEmail
- subject
- bodyText
- bodyHtml if available
- providerMessageId
- threadId if available
- receivedAt
- rawPayload safely if useful
- status: received, classified, routed, failed

Use existing migration/schema patterns.

3. Classification layer
Create:

server/services/agentmail-inbound-router.ts

Implement:
- classifyInboundEmail()
- routeInboundEmail()
- processInboundAgentMail()
- mapInboxToDefaultAgent()
- extractIntentSignals()

Classification categories:
- new_lead
- booking_request
- reschedule_request
- cancellation_request
- pricing_question
- employment_candidate
- support_issue
- billing_issue
- athlete_parent_question
- coach_partner_inquiry
- software_bug_report
- urgent_escalation
- general_question

Use deterministic rules first:
- hiring@ → employment_candidate unless clearly support/spam
- scheduling@ → booking/reschedule/cancellation
- revenue@ → new lead, pricing, partner inquiry
- support@ → support issue, billing issue, bug report
- ceo@ / operations@ → urgent escalation, partner inquiry, operational issue

Then optionally use the existing AI/agent inference pattern if available.

4. Routing behavior
Based on classification, create the right downstream action:

new_lead:
- create or update lead/prospect/applicant record if matching tables exist
- add to Revenue Recovery Queue or Attention Inbox
- assign Revenue Agent

booking_request / reschedule_request / cancellation_request:
- create scheduling attention item
- assign Scheduling Agent
- include proposed date/time/location details if extracted

employment_candidate:
- create employment applicant if table exists
- assign Hiring/Employment Agent
- include resume/experience/location signals if present

support_issue / billing_issue / software_bug_report:
- create support attention item
- assign Support/Client Success Agent
- if software_bug_report, optionally create software improvement task

urgent_escalation:
- create high severity Attention Inbox item
- notify CEO Heartbeat / operations agent

general_question:
- create medium/low severity Attention Inbox item with suggested reply

spam_or_noise:
- store only; do not route unless admin manually reviews

5. Attention Inbox integration
Add inbound email items to the existing Attention Inbox system.

Each item should include:
- source: agentmail
- sourceMessageId
- inbox
- classification
- severity
- recommendedAction
- suggestedReply if generated
- routedAgent
- status: open

- support issues

Store the suggested reply in action_payload or an appropriate field.
