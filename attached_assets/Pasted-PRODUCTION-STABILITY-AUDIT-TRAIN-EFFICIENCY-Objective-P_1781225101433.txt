PRODUCTION STABILITY AUDIT — TRAIN EFFICIENCY

Objective:
Perform a full production stability audit focused on runtime reliability, user-facing failures, error handling, deployment safety, and business-critical workflows.

This audit is separate from TypeScript correctness. Assume the codebase now passes typecheck with zero errors.

──────────────────────────────
PHASE 1 — APPLICATION CRASH PROTECTION
──────────────────────────────

Audit all React entry points and determine whether runtime exceptions can white-screen the application.

Tasks:

1. Verify global Error Boundary coverage.
    * App root
    * Authenticated app shell
    * Public landing pages
    * Admin pages
    * Org portal pages
2. Identify pages capable of crashing the entire React tree.
3. Add route-level Error Boundaries where appropriate.
4. Ensure all Error Boundaries:
    * Log errors
    * Display recovery UI
    * Include retry actions
    * Prevent application-wide crashes

Return:

* Existing coverage
* Missing coverage
* Recommended implementation locations

──────────────────────────────
PHASE 2 — REACT QUERY RESILIENCE
──────────────────────────────

Audit all React Query usage.

Tasks:

1. Identify queries missing:
    * Error handling
    * Loading states
    * Empty states
2. Identify mutations lacking:
    * Error toast
    * Retry strategy
    * Failure visibility
3. Identify query chains that can break if one request fails.
4. Verify:
    * Query invalidation correctness
    * Cache update safety
    * Optimistic update rollback behavior

Return:

* High-risk queries
* High-risk mutations
* Missing loading/error states

──────────────────────────────
PHASE 3 — AUTHENTICATION & SESSION RESILIENCE
──────────────────────────────

Audit:

* Coach authentication
* Admin authentication
* Org authentication
* Portal authentication

Tasks:

1. Verify expired sessions redirect safely.
2. Verify unauthorized requests do not crash pages.
3. Verify:
    * Missing org token
    * Invalid org token
    * Expired session cookie
    * Permission denial

all produce safe UI behavior.

4. Verify all protected routes.

Return:

* Broken auth flows
* Missing guards
* Pages that assume authentication incorrectly

──────────────────────────────
PHASE 4 — API FAILURE AUDIT
──────────────────────────────

Simulate API failures.

For each critical page:

* Network timeout
* 500 error
* Empty response
* Malformed response

Verify:

1. No runtime crashes occur.
2. User sees actionable feedback.
3. Loading state exits correctly.
4. Retry path exists.

Audit:

* Scheduling
* Command Center
* Athlete Management
* Lead Capture
* Billing
* Education
* Intelligence
* Notifications
* AI Departments

Return:

* Failure points
* Missing fallbacks
* Crash scenarios
