# Goal

Make AgentMail learning visible, editable, and coach-controllable.

Right now, AgentMail learns from edits, rejections, and regeneration feedback, but coaches cannot easily see:

* What the AI has learned
* Which rules are active
* Which communication domain a rule belongs to
* Which rules are helping or hurting
* How to teach the AI proactively

This phase turns the hidden learning system into a transparent **AgentMail Learning Center**.

# Scope

Create an admin UI where coaches can view learned rules.

Suggested route:

```txt
/admin/agentmail-learning
```

or a new tab inside existing AgentMail Admin.

Show rules from:

```txt

```

Group/filter by communication domain:

* athlete_lead
* parent_lead

* onboarding
* retention
* payment_recovery
* program_assignment
* win_back

Display for each rule:

* Rule text

* Confidence score

* Last reinforced / updated date
* Created date

* Toggle active/inactive

* Delete/archive rule if safe
* Filter by type/domain/status/search

Do not expose this to non-admin users.

## 2. Standing Instructions / Teach the AI

Add a way for coaches to manually create organization-level communication rules.

Suggested table if none exists:

```sql
agent_draft_coaching_rules (
  id varchar primary key default gen_random_uuid(),
  org_id varchar not null,
  communication_domain varchar not null default 'general',
  rule_type varchar not null default 'instruction',
  rule_text text not null,
  authored_by varchar not null,
  is_active boolean not null default true,
  created_at timestamp default now(),
  updated_at timestamp default now()
)
```

Purpose:

These are direct coach-authored instructions, separate from inferred learned rules.

Examples:

* “For lead outreach, always mention the athlete’s sport in the first paragraph.”
* “For parent emails, keep messages under 150 words.”
* “Never mention pricing until after the evaluation.”
* “Use a warm, direct coaching tone.”
* “End scheduling emails with one clear next step.”

UI:

* Add instruction
* Choose domain
* Choose type
* Save
* Toggle active/inactive
* Edit
* Delete/archive

## 3. Merge Standing Instructions Into Learning Context

Update:

```txt
getMessageLearningContext()
```

So it returns both:

1. Learned rules from `agent_message_learning_rules`
2. Coach-authored standing instructions from `agent_draft_coaching_rules`

Ordering:

* Standing instructions first
* High-confidence learned rules second
* Recent/reinforced learned rules next

If there are too many:

* Limit total length
* Prefer active standing rules
* Prefer high-confidence learned rules
* Prefer domain-specific rules over general rules

Rules:

* Domain-specific context should include:

  * active rules for that exact domain
  * active global/general rules if appropriate
* Inactive rules must not be included.
* Deleted/archived rules must not be included.

# Data Safety

* Never allow one org to view another org’s rules.
* Do not delete learning history unless existing patterns allow hard delete.
* Prefer soft-disable / archive over destructive delete.
* Do not expose internal prompts.
* Do not expose private athlete/guardian data in rules.
* Sanitize user-authored instruction text.
* Keep rule text concise.
