Build Notification Automation Hooks for TrainEfficiency.

GOAL:
Transform the Communication + Notification Layer from a passive inbox into an active event-driven operating system.

The system should automatically generate notifications and messages from existing platform events.

CONTEXT:
The following systems already exist:
- PR Tracker
- Workout Builder
- Workout Execution + Readiness
- Athlete Intelligence
- Watchlists
- Coach Alerts
- Messaging System
- Notification Center

This phase connects those systems together.

====================================================
1. AUTOMATION ENGINE
====================================================

Create a lightweight internal automation/event hook layer.

Purpose:
- listen for important system events
- generate notifications/messages automatically
- centralize notification logic
- avoid duplicated notification code everywhere

Create:

server/services/notification-automation.ts

Core helper:
triggerNotificationEvent(eventType, payload)

This service should:
- validate payload
- determine recipients
- generate notification(s)
- optionally generate coach alerts
- deduplicate repeated events where appropriate

====================================================
7. DEDUPLICATION RULES
====================================================

Avoid spam.

Examples:
- do not send 5 low-readiness alerts in one day
- combine repeated missed-workout events
- avoid duplicate PR celebration spam

Implement:
- cooldown windows
- grouped alerts where appropriate
