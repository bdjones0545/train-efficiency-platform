Build Phase 3 — Scheduling Agent + Google Calendar Intelligence

Objective:

Now that Composio Google Calendar is fully operational and validated, integrate it directly into the Scheduling Agent so calendar intelligence becomes part of the booking workflow.

The goal is for the Scheduling Agent to move beyond static availability and become capable of reasoning from real Google Calendar data while still preserving approval gates for write actions.

# Feature 1 — Availability Intelligence

Scheduling Agent should use:

```text
POST /api/composio/calendar/availability
```

before suggesting times.

Workflow:

User requests appointment.

Agent:

1. Checks Google Calendar availability.
2. Finds open blocks.
3. Applies business rules.
4. Returns best options.

Example:

```text
Available:

Tuesday 10:00 AM
Tuesday 11:00 AM
Wednesday 2:00 PM

Recommended:
Tuesday 10:00 AM
```

---

# Feature 2 — Conflict Detection

Before creating any booking:

Agent must check:

```text
GET /api/composio/calendar/events
```

Determine:

* overlaps
* double bookings
* insufficient buffer time

Return:

```text
Conflict detected
```

instead of creating request.

# Feature 4 — Event Creation Workflow

When a booking is accepted:

Agent does NOT create event directly.

Instead:

```text
calendar_create_event
```

creates approval request.

Flow:

Agent
→ Queue
→ Human Approval
→ Google Calendar Event
→ Outcome Log

Reuse existing approval architecture.
