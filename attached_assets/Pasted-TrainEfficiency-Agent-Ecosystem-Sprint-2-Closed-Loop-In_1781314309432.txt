TrainEfficiency Agent Ecosystem — Sprint 2: Closed-Loop Intelligence & Unified Action System

Objective

Sprint 1 established safety, auditability, and real system health.

Sprint 2 converts Hermes from a passive learning engine into an active intelligence participant that can generate reviewable recommendations and participate directly in CEO Heartbeat decision-making.

The goal is to create a closed operational loop:

Gmail Sync
→ Conversation Intelligence
→ Hermes Analysis
→ Action Recommendation
→ Human Review
→ Approved Execution
→ Audit Trail
→ CEO Heartbeat
→ System Health

Deliverable 2 — Hermes Recommendation Engine

Current State

Hermes creates learning insights but cannot create actions.

Required

Create:

server/services/hermes-recommendation-engine.ts

Hermes should evaluate:

* Gmail conversations
* Opportunity replies
* AgentMail outcomes
* Lead conversion history
* Workflow failures
* Outreach performance

Generate structured recommendations.

Example:

{
type: “follow_up”,
title: “Follow up with Coach Mike”,
reason: “High engagement detected, no response in 5 days”,
confidence: 0.91,
sourceSystem: “gmail”,
sourceConversationId: “…”,
recommendedAction: “schedule_call”
}

Store in:

autonomous_action_queue

with:

source_system = ‘hermes’

Deliverable 4 — Unified Action Queue

Current Problem

Four separate action queues exist:

* gmail_agent_actions
* autonomous_action_queue
* agent_pending_actions
* in-memory daily queue

Required

Create:

server/services/unified-action-queue.ts

Single read layer.

Aggregate:

* Gmail actions
* Hermes recommendations
* Autonomy actions
* Scheduling actions

Return normalized objects:

{
id,
sourceAgent,
sourceSystem,
type,
title,
confidence,
approvalRequired,
status,
createdAt
}

No UI changes yet.

Create unified service first.

Deliverable 5 — Action Review Context

Current Problem

Required

/api/actions/:id/context

This endpoint becomes the foundation for future approval UI.
