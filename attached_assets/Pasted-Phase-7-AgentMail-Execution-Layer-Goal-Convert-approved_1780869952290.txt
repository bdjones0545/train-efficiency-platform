Phase 7 — AgentMail Execution Layer

Goal:
Convert approved outreach drafts into real outbound conversations using AgentMail.

This phase introduces outbound execution while maintaining strict human approval controls.

Important:

* No fully autonomous outreach.
* No automatic sending.
* No auto-approval.
* No self-applying behavior.
* Human approval required before every send.

The objective is:

Discovery
→ Qualification
→ Outreach Draft
→ Approval
→ Send
→ Conversation

⸻

BACKEND

Create:

server/services/opportunity-outreach-execution-agent.ts

Primary function:

executeOutreachDraft(orgId, draftId)

Responsibilities:

1. Validate draft
2. Confirm draft status = approved
3. Resolve recipient information
4. Send through AgentMail
5. Persist execution record
6. Log agent events
7. Update pipeline status

OUTREACH EXECUTION LOGIC

Rules:

Only approved drafts can be sent.

If draft status != approved:

Return 400.

When sent successfully:

Update:

opportunity_outreach_drafts.status = sent

Update:

opportunity_acquisition_opportunities.status = contacted

Create execution record.

Create agent event.

SAFETY CONTROLS

Human approval required.

Human click required.

No automatic sends.

No scheduled sends.

No autonomous execution.

One draft = one manual send action.

⸻

SUCCESS CRITERIA

* Approved drafts can be sent
* Execution records are stored
* Pipeline automatically updates to Contacted
* Executions tab exists
* Agent events are logged
* AgentMail is integrated
* Delivery metadata is stored
* Reply tracking fields exist
