REPLIT PROMPT — EMAIL HISTORY AS TEAM OUTREACH COMMAND CENTER

Objective:
Upgrade the “Email History” page into the management hub for the AI Email Agent.

This page should manage:
1. Athletic team prospect discovery
2. Daily outbound outreach
3. Email history
4. AI-generated drafts
5. Sent emails
6. Replies/status tracking
7. Safety rules like cooldowns, opt-outs, and daily caps

Core behavior:
The system should automatically build a local prospect list of athletic teams around our service area and send up to 10 outreach emails per day to qualified prospects.

Important:
Do not spam.
Do not send more than 10 prospecting emails per day per organization.
Do not contact the same prospect more than once inside the cooldown window.
Respect do-not-contact, opt-outs, and invalid/missing emails.

PAGE TO MODIFY:
Email History page/sidebar route.

Rename page heading to:
“Email Agent”

Subtitle:
“Manage outreach, prospects, sent emails, and AI-generated follow-ups.”

PHASE 2 — PROSPECT DISCOVERY

Create or extend a backend service:

server/team-prospecting/team-prospect-discovery.ts

Purpose:
Find and store athletic team prospects around the organization’s area.

Prospect fields:

team_training_prospects:
- id
- organization_id
- name
- school_or_org_name
- sport
- level
  - middle_school
  - high_school
  - club
  - college
  - adult
  - other
- contact_name nullable
- contact_title nullable
- contact_email nullable
- contact_phone nullable
- website_url nullable
- source_url nullable
- city nullable
- state nullable
- distance_miles nullable
- confidence_score numeric
- estimated_value numeric
- status enum:
  - discovered
  - queued
  - drafted
  - sent
  - replied
  - interested
  - not_interested
  - do_not_contact
  - invalid_contact
- last_contacted_at nullable
- next_eligible_contact_at nullable
- notes nullable
- created_at
- updated_at

Discovery rules:
- Use organization city/state as the center point.
- Search for athletic teams, schools, clubs, and programs nearby.
- Prioritize:
  - high school athletic teams
  - club sports teams
  - travel teams
  - private academies
  - athletic directors
  - coaches with public email addresses
- Store source_url for every discovered contact.
- Never invent contact info.
- If email is not found, store the prospect but mark contact_email as null.
- Only prospects with valid contact_email can enter the outreach queue.

Add a button on Email Agent page:
“Find Team Prospects”

When clicked:
- Run prospect discovery
- Save prospects to database
- Show progress state
- Return number found, number with emails, number skipped as duplicates

PHASE 3 — DAILY EMAIL QUEUE

Create service:

server/email-agent/daily-team-outreach-queue.ts

Function:
buildDailyTeamOutreachQueue(organizationId)

Rules:
- Select up to 10 prospects per day
- Only include prospects where:
  - contact_email is present
  - status is discovered, drafted, or queued
  - do_not_contact is false
  - email_opt_out is false
  - last_contacted_at is null OR older than cooldown window
- Prioritize by:
  1. confidence_score
  2. estimated_value
  3. distance_miles closest
  4. high school / club / travel programs first
- Mark selected prospects as queued

Daily limit:
- max_team_outreach_emails_per_day = 10
- Count all sent team outreach emails today for the organization before sending.
- If already at 10, block additional auto-sends and show:
  “Daily outreach limit reached.”

PHASE 5 — AUTO-SEND SYSTEM

Create service:

server/email-agent/team-outreach-auto-send.ts

Function:
runDailyTeamOutreachSend(organizationId)

Behavior:
- Build or load today’s queue
- Generate drafts if missing
- Send up to 10 emails per day
- Use existing SendGrid integration
- Log every sent email
- Update prospect status to sent
- Set last_contacted_at = now
- Set next_eligible_contact_at based on cooldown

Safety:
Before every send, check:
- daily count < 10
- prospect has valid email
- prospect not do_not_contact
- prospect not opted out
- prospect not contacted inside cooldown
- no duplicate sent email to same email address within cooldown
- organization email sending enabled
