========================
PHASE 1 — SAFE AUTO-SEND ENGINE
========================

Extend autoExecuteMode (levels 0–3):

Level 0: OFF (manual)
Level 1: Draft only (current)
Level 2: Semi-auto (draft + suggest)
Level 3: AUTO-SEND (new)

At Level 3, allow automatic sending ONLY for:

Allowed:
- follow-up messages (after 24h+ no response)
- backfill outreach for slots within next 24h
- package renewal reminders

Blocked:
- first-touch churn outreach
- upsell messages above $X threshold (configurable)
- any booking creation

Rules:
- must check actionScore threshold before sending
- must log as autoSent = true
- must include “autoReason” (why it was sent automatically)

Create:
execute_auto_actions()

Runs every 30–60 minutes:
- scans eligible actions
- filters by safety rules
- executes send_client_message
- updates action status to "sent_auto"

========================
PHASE 2 — TIME-BASED OPTIMIZATION
========================

Create system:
compute_time_performance_profile

For each actionSubType AND hour of day:
- messages sent
- responses
- bookings
- conversion rate by hour

Store:
timePerformanceProfile

Agent rules:
- When drafting or sending:
  → prefer highest-converting time window

Example:
- Backfill converts best at 5–7pm → delay/send in that window

Agent must say:
“I’m scheduling this for 6pm — your highest conversion window for this type.”

Create:
messagePerformanceProfile

========================
PHASE 4 — MULTI-STEP CAMPAIGN ENGINE
========================

Create campaign system:

campaignTemplates:

- package_renewal (2-step)

Create:

- currentStep
- nextActionAt

Engine:

- executes or drafts next message
