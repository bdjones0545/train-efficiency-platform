Build Persistent Athlete Intelligence Layer (PAIL)

Objective:

TrainEfficiency’s communications intelligence, CEO Heartbeat, governance, outreach, learning loops, and outcome intelligence are now highly mature.

The next evolution is to bring the same architecture to athlete development.

Current State:

Athlete Context Broker
→ TrainChat
→ Program Generated

Desired State:

Athlete Memory
→ Context Broker
→ TrainChat
→ Session Execution
→ Biofeedback
→ Adaptation Engine
→ Outcome Intelligence
→ Athlete Learning
→ Future Program Decisions

TrainChat should evolve from a program generator into the central training intelligence layer.

⸻

Phase 1 — Persistent Athlete Memory

Create:

athlete_memory_profiles

Purpose:

Long-term athlete intelligence that survives across every workout, program, season, and coach interaction.

Fields:

* id
* org_id
* athlete_id

Training Identity

* primary_sport
* secondary_sport
* position
* competition_level
* training_age_years

Athlete Preferences

* preferred_exercises
* disliked_exercises
* preferred_session_length
* preferred_training_days

Movement Intelligence

* movement_restrictions
* recurring_compensations
* technical_focus_areas
* coaching_cues_that_work

Readiness Intelligence

* normal_readiness_range
* fatigue_patterns
* recovery_patterns
* stress_patterns

Adaptation Intelligence

* exercises_that_progress_well
* exercises_that_stall

Coach Intelligence

Learning Metadata

* memory_confidence

* last_synthesized

⸻

Phase 2 — Athlete Learning Engine

Create:

server/services/athlete-learning-engine.ts

Purpose:

Continuously learn from athlete behavior.

Inputs:

* workout execution
* readiness logs
* compliance
* PRs
* missed sessions
* coach notes
* pain reports
* exercise substitutions

Outputs:

Updates athlete_memory_profiles.

Examples:

Athlete consistently misses Friday sessions
→ Recommend removing Friday as a training day
Athlete responds well to unilateral work
→ Increase future exposure
Athlete repeatedly swaps back squats
→ Learn preference

Phase 5 — Coach Note Intelligence

Coach notes are currently underutilized.

Build:

coach-note-intelligence-service.ts

Every coach note should be analyzed.

Extract:

* movement issues
* motivation patterns
* confidence issues
* technical weaknesses
* recurring themes

Update athlete memory automatically.

Example:

Three coach notes mention:
"slow first step"
Memory updated:
Acceleration weakness detected

⸻

Phase 10 — Athlete Autonomy Progression

Create athlete-level trust scoring.

As the system learns:

Level 0
Manual coach review.

Level 1
Suggest adaptations.

Level 2
Auto-adjust low-risk training variables.

Level 3
Autonomous micro-adjustments with coach monitoring.

Never allow:

* injury diagnosis
* medical recommendations
* major program rewrites

without coach approval.

* Training decisions become outcome-driven.
* Athlete learning compounds over time.
