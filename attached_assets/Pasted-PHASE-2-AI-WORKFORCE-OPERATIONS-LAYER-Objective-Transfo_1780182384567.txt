PHASE 2 — AI WORKFORCE OPERATIONS LAYER

Objective

Transform the AI Workforce Wizard from a configuration system into a measurable operational system.

Phase 1 successfully persisted workforce settings, governance policies, department enablement, workflow provenance, and integration validation.

Phase 2 should focus on making the AI workforce observable, measurable, configurable, and actionable.

Do not redesign existing functionality. Extend and operationalize it.

PART 1 — Replace Placeholder Agent Statistics

Audit all workforce dashboard endpoints and identify any remaining placeholder, mocked, or Math.random() generated values.

Specifically inspect:

/api/workforce/agent-stats/:agentId

Current issue:
actionsToday and approvalsNeeded are generated using Math.random().

Required:

Replace all synthetic values with live calculations.

Data sources should include:

unified_agent_action_log

integration_execution_log

workflow_jobs

org_activity_events

For each agent return:

Return only real values.

No estimated or random metrics.

PART 11 — Verification Requirements

Before implementation:

Identify every existing source of workforce telemetry.

Do not duplicate metrics already available elsewhere.

Reuse existing logging infrastructure whenever possible.

Avoid introducing redundant tables.

PART 12 — Success Criteria

At completion the AI Workforce system should:

Show real operational metrics.

Show real agent activity.

Show real approval queues.

Provide workforce configuration management.

Provide workforce audit history.

Provide readiness guidance.

Provide integration setup guidance.

Provide agent performance monitoring.

Provide organization-level workforce intelligence.

Eliminate all placeholder and synthetic workforce statistics.

The result should feel like a true AI Workforce Operating System rather than a setup wizard.
