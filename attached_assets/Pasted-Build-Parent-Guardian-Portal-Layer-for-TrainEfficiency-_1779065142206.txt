Build Parent / Guardian Portal Layer for TrainEfficiency.

GOAL:
Add parent/guardian accounts linked to athletes so families can monitor athlete participation, progress, communication, education completion, and scheduling.

This is especially important for:
- youth sports
- high school programs
- semi-private coaching
- facilities with minors

The Parent Portal should increase:
- accountability
- communication clarity
- retention
- professionalism
- family engagement

IMPORTANT:
Parents are observers/supporters.
They should NOT have unrestricted athlete control.

====================================================
1. CORE CONCEPT
====================================================

Parent accounts can:
- view linked athlete activity
- receive notifications
- monitor progress
- view schedules
- view education completion
- message coaches later (future phase)

Parents cannot:
- edit workouts
- edit PRs
- impersonate athletes
- access coach-private notes
- access hidden intelligence data
- modify readiness data

====================================================
2. DATABASE TABLES
====================================================

Create:

parent_guardians
- id
- orgId
- orgUserId
- relationshipType
  (mother, father, guardian, other)
- createdAt

athlete_guardian_links
- id
- orgId
- athleteUserId
- guardianUserId
- status
  (pending, active, revoked)
- invitedByUserId
- createdAt
- activatedAt nullable

guardian_notifications
- id
- orgId
- guardianUserId
- athleteUserId
- type
  (attendance, workout_completion,
   missed_workout, education_progress,
   coach_announcement, pr_update)
- title
- message
- metadata jsonb
- isRead
- createdAt

====================================================
3. AUTHENTICATION
====================================================

Reuse existing:
- org_users
- org_memberships
- org_sessions

Add new role:
guardian

Guardians log in through the same org auth system.

====================================================
4. GUARDIAN INVITE FLOW
====================================================

Coaches or athletes can:
- invite guardian by email

Flow:
1. Send invite
2. Guardian creates account
3. Guardian linked to athlete
4. Guardian receives portal access

Add invite status:
- pending
- accepted
- revoked

====================================================
5. GUARDIAN PORTAL
====================================================

Create route:

/org/:slug/guardian

Guardian sees:

A. Linked Athletes
- athlete cards
- team
- sport
- progress summary

B. Schedule
- upcoming bookings
- recent attendance
- missed sessions

C. Workout Activity
- assigned workouts
- completion %
- recent sessions completed

D. PR Progress
- recent PRs
- milestone badges
- progress highlights

E. Education Progress
- module completion %
- pathway progress
- overdue modules

F. Notifications
- coach announcements
- missed workout alerts
- PR celebrations
- schedule reminders

====================================================
6. COACH CONTROLS
====================================================

Coach can:
- view guardian links
- invite/remove guardians
- control guardian permissions
- choose visible categories

Examples:
- allow PR visibility
- allow attendance visibility
- hide readiness scores
- hide private athlete notes

====================================================
7. PERMISSION LAYER
====================================================

Guardians CAN view:
- schedules
- attendance
- workout completion
- education progress
- PR milestones
- team announcements

Guardians CANNOT view:
- private coach notes
- readiness scores/details
- internal intelligence alerts
- AI recommendations
- hidden athlete metadata
- other athletes

====================================================

Create:

POST /api/org/guardians/invite
