Redesign the My Bookings page to improve organization, visibility, and booking type coverage.

Target page:
My Bookings

Current issues:
1. The page appears to only show Semi-Private sessions.
2. It should also show:
   - 1-on-1 sessions
   - group sessions
   - assessments
   - recovery sessions
   - team training sessions if applicable
3. Cards are too description-heavy.
4. Bookings should be easier to scan by coach, date, and session type.

Goal:
Make My Bookings a clear training schedule hub.

--------------------------------------------------
PHASE 1 — DATA COVERAGE AUDIT
--------------------------------------------------

Inspect the current My Bookings data source and confirm why only Semi-Private sessions appear.

Check:
- API route used by My Bookings
- booking/session query filters
- session type filters
- status filters
- coach/user ownership filters
- whether 1-on-1 sessions are stored in a different table or endpoint
- whether group/open sessions use a separate data model

Required:
My Bookings should include all sessions tied to the current user/athlete/client.

Include:
- 1-on-1
- Semi-Private
- Group
- Team Training
- Assessment
- Recovery
- Completed
- Cancelled
- Past

Do not duplicate records.

--------------------------------------------------

Primary details:

Time
Coach
Location
Sport / Focus

Coach Hunter Thaxton

Track & Field · Advanced

[Reschedule] if allowed
[Cancel] if allowed

Description:

Show only a short 1-line summary if needed.
Full description belongs inside details drawer/modal.
