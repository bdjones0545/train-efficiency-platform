AI Comms Center — Auto-Eligible Draft UX Upgrade

Objective:

Now that ActionExecutor correctly routes policy-approved emails into the approval queue instead of silently dropping them, make those drafts visually distinct from drafts that still require full review.

Problem

Today all pending drafts look identical.

The user cannot quickly tell the difference between:

* Drafts that passed all governance checks and are safe to send
* Drafts that still require full human review

This creates unnecessary friction and hides one of the most valuable outputs of the autonomy engine.

⸻

Auto-Eligible Badge

When:

result.autoExecuteEligible === true

Display:

🟢 Policy Approved
Needs Send Confirmation

or

🟢 Auto-Eligible
Passed Governance Checks

Badge color:

* Green
* High contrast
* Always visible on card header

⸻

Draft Priority Grouping

Add three sections:

Auto-Eligible

Criteria:

result.autoExecuteEligible === true

Description:

Passed all governance checks.
Ready to send with one click.

⸻

Awaiting Review

Criteria:

approvalRequired === true
&& !result.autoExecuteEligible

Description:

Requires human review before sending.

⸻

Blocked

Criteria:

status === "blocked"

Description:

Blocked by governance policy.

Show block reason.

⸻

One-Click Send

For Auto-Eligible drafts:

Replace:

Approve
Edit
Reject

With:

Send Now
Review First

Send Now:

* executes existing approval/send path
* creates outcome row
* writes timeline event
* writes audit log
* preserves all existing governance protections

No additional review required.

Future-Proofing

Store:

result.autoExecuteEligible

as first-class fields in UI models.

Acceptance Criteria

* Auto-eligible drafts are visually distinct
* Green governance-approved badge visible
* Auto-eligible section appears at top of queue
* One-click Send Now action available
* KPI card shows count
* Domain dashboards show Auto-Eligible Rate
* Existing approval workflow unchanged
* Existing outcome tracking unchanged
* Existing audit logging unchanged
