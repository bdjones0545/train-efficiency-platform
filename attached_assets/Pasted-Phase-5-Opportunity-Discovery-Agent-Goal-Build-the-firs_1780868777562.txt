Phase 5 — Opportunity Discovery Agent

Goal:
Build the first autonomous Opportunity Discovery Agent that finds opportunities, filters them, deduplicates them, stores them, and feeds the existing Qualification + Outreach workflow.

Important:
Do NOT build email sending.
Do NOT build automated outreach.
Do NOT build auto-apply behavior.

The objective is:

Discovery
→ Qualification
→ Outreach Draft

using real discovered opportunities.

⸻

BACKEND

Create:

server/services/opportunity-discovery-agent.ts

Primary function:

runOpportunityDiscovery(orgId)

Responsibilities:

1. Discovery
    Fetch opportunities from configured sources.

Initial architecture should support:

* LinkedIn Jobs
* Indeed
* TeamWork Online
* HigherEdJobs
* NCAA Careers
* Company Career Pages

The implementation can initially use mock providers, RSS feeds, APIs, or pluggable adapters.

The important part is the architecture.

Create:

interface OpportunitySourceAdapter {
name: string;
discover(): Promise<DiscoveredOpportunity[]>;
}

Each provider should be isolated so future integrations are easy.

DEDUPLICATION

Create fingerprinting.

Generate fingerprint from:

* title
* company
* location

Hash the normalized values.

Before creating an opportunity:

Check for existing fingerprint.

Skip duplicates.

Track duplicate count.

FILTERING

Positive signals:

* remote
* online
* virtual
* coach
* coaching
* trainer
* performance
* strength
* conditioning
* wellness
* consultant
* programming

Negative signals:

* onsite only
* relocation required
* field technician
* travel required
* warehouse
* delivery

Create configurable filtering logic.

Only store opportunities passing filters.
