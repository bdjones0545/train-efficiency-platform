# TrainEfficiency Replit Task

## AgentMail Phase A — Close the Learning Loop

You are working inside the TrainEfficiency codebase.

The AgentMail Draft Review & Learning audit found that the learning architecture already exists, but it is not applied consistently to the highest-volume draft generation paths.

The core problem:

> Coaches edit, reject, and regenerate drafts. The system stores learning rules. But most new drafts do not use those rules.

This phase fixes that.

---

# Goal

Make future AgentMail drafts improve from prior coach feedback.

AgentMail should use organization-specific and communication-domain-specific learning context when generating:

* Athlete lead outreach
* Guardian/parent welcome drafts
* Hot lead evaluation CTAs
* Follow-up sequences
* Workflow-triggered emails
* Welcome emails
* Retention emails
* Win-back emails
* Payment recovery emails
* Program assignment emails

---

# Scope

## 1. Wire Learning Context Into Lead Intake Drafts

Update:

```txt
server/services/intelligent-lead-intake-service.ts
```

Before generating each draft, call:

```ts
getMessageLearningContext(orgId, communicationDomain)
```

Inject that learning context into the system prompt.

Important domains:

* athlete lead outreach → `athlete_lead`
* parent/guardian outreach → `parent_lead`
* hot lead evaluation CTA → `evaluation_scheduling`
* general lead follow-up → `lead_followup`

Rules:

* If no learning context exists, generation should behave exactly as before.
* If learning context exists, it should guide tone, CTA, length, personalization, and avoid-rules.
* Learning context should be additive, not destructive.
* Do not expose internal learning text to the athlete/parent.

# 6. Capture Implicit Learning From Coach Edits

Verify that edited subject/body are captured whenever a coach approves with edits.

If already implemented, document where.

If incomplete:

* Ensure original subject/body are preserved.
* Ensure edited subject/body are stored.
* Ensure the feedback extraction service receives both original and edited versions.
* Ensure subject changes are included, not only body changes.
* Ensure communication domain is stored with the feedback record.

Goal:

> Every meaningful coach edit becomes a learning signal, even if the coach does not click a feedback chip.
