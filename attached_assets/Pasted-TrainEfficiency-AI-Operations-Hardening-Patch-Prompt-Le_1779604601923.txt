PART 1 — Emergency pause guard for legacy send paths

Patch:

* server/email.ts
* server/sms.ts

Before any real outbound send:

* resolve orgId if available
* check org_ai_governance_settings.emergencyPauseEnabled
* if paused:
    * block the send
    * return a clear error
    * write to unified_agent_action_log
    * write/send failure to communication_log if applicable

Message:
“Blocked: AI operations are paused for this organization. Emergency pause must be disabled before outbound communication can be sent.”

Requirements:

* Do not crash if orgId is missing
* If orgId is missing, log warning and continue only for system-level non-org emails
* For org-scoped sends, emergency pause must be enforced
* Keep current email/SMS behavior otherwise unchanged
