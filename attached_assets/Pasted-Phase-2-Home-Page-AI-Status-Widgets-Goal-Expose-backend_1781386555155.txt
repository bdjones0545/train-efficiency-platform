Phase 2 — Home Page AI Status Widgets

Goal

Expose backend intelligence on the Home page so users can immediately see what the AI workforce is doing without opening multiple admin pages.

Do not change backend systems.
Do not remove routes.
Do not move data ownership.

Only surface existing intelligence already available in the platform.

Core Principle

The backend is already working.

The Home page should answer:

* What is the AI workforce doing?
* What needs my attention?
* What happened recently?
* What should I do next?

without requiring navigation into Hermes, CEO Heartbeat, AgentMail, Workflow Builder, Governance, or Organizational Memory.

AI Workforce

Visibility Rules

Do NOT expose:

* raw governance controls
* workflow graphs
* execution internals
* developer diagnostics
* infrastructure configuration

Expose:

* outcomes
* recommendations
* activity
* memory
* approvals
* health

Acceptance Criteria

* Users can understand platform activity from the Home page alone.
* Home page becomes the operational dashboard.
* Existing backend systems gain visibility.
* No backend logic changes.
* No route removals.
* No duplicate pages created.
* All cards use real persisted data.
* Empty states explain why data is unavailable.
* TypeScript passes.
* Server starts cleanly.

Bonus Audit

For each widget added, document:

Widget | Endpoint | Backend Source | Database Tables | Refresh Strategy

Ensure every widget has a clearly defined source of truth.
