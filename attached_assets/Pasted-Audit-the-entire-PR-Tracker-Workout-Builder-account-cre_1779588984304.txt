Audit the entire PR Tracker / Workout Builder account creation and onboarding pipeline to ensure users are properly integrated into the organization system and receive automated onboarding communication.

Current flow:
Users can create:
- Athlete accounts
- Team Coach accounts

through the PR Tracker / Workout Builder auth modal.

We now need to verify:
1. how these accounts connect into the org ecosystem
2. whether onboarding/intro communication is sent automatically
3. whether org admins/coaches can immediately see/manage these users
4. whether permissions and org memberships are assigned correctly

PART 1 — Full onboarding audit

Trace the full signup flow from:
PR Tracker / Workout Builder modal
→ account creation
→ org association
→ role assignment
→ onboarding communication
→ dashboard visibility

Audit:
- frontend form submission
- API route
- database writes
- auth provider sync
- org membership creation

- notification pipeline
- email automation pipeline

PART 2 — Verify org integration

For every signup type verify:

ATHLETE:
- user record created
- org membership created
- role assigned correctly
- optional team join code processed
- appears in org athlete/user management
- appears in coach dashboard
- can access athlete-facing tools only

TEAM COACH:
- user record created
- org membership created
- role = coach
- appears in org staff/team management
- receives coach permissions
- can access PR Tracker management + workout builder tools

Verify:
- no orphaned users
- no missing org linkage
- no duplicate memberships
- no fallback guest accounts
