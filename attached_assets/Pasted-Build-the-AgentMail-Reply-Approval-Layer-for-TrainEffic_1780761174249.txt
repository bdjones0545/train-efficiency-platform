Build the AgentMail Reply Approval Layer for TrainEfficiency.

Goal:
TrainEfficiency agents can draft replies to inbound emails, but a human remains in control. Every suggested response should move through:

Inbound Email
→ Classification
→ Suggested Reply Draft
→ Human Review
→ Approve/Edit
→ Send via AgentMail
→ Outcome Tracking
→ Agent Learning Metrics

Inspect first:
- Existing AgentMail integration
- agentmail-service.ts
- agentmail-inbound-router.ts
- agent_mail_messages table
- agent_mail_inbound_messages table
- Attention Inbox
- CEO Heartbeat
- Software Improvement Tasks
- Agent Operating Timeline
- Existing agent outcome/effectiveness tracking systems

Do not duplicate patterns that already exist.

--------------------------------------------------
PHASE 2 — Draft Creation
--------------------------------------------------

When inbound emails generate suggested replies:

Automatically create a reply queue record.

Supported classifications:

- new_lead
- pricing_question
- booking_request
- reschedule_request
- employment_candidate
- support_issue
- athlete_parent_question
- coach_partner_inquiry
- general_question

Store:
- original AI draft
- confidence
- classification
- assigned agent

No email should be sent automatically.

--------------------------------------------------
PHASE 3 — Approval Workflow
--------------------------------------------------

Create API routes:

GET /api/agentmail/replies
GET /api/agentmail/replies/:id
PATCH /api/agentmail/replies/:id
POST /api/agentmail/replies/:id/approve
POST /api/agentmail/replies/:id/reject
POST /api/agentmail/replies/:id/send

Approval:
- approve stores approver + timestamp
- reject stores reason
- edit updates edited_body
- send uses edited_body if present, otherwise draft_body
