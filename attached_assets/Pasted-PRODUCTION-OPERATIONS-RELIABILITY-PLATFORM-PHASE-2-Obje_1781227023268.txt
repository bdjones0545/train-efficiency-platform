PRODUCTION OPERATIONS & RELIABILITY PLATFORM — PHASE 2

Objective:

──────────────────────────────
TASK 1 — STRUCTURED LOGGING LAYER
──────────────────────────────

Current State:

Frontend crashes, query failures, API errors, and agent failures are logged but mostly exist in console output.

Goal:

Persist critical operational events.

Create:

system_logs
client_errors
query_failures

tables.

──────────────────────────────
system_logs
──────────────────────────────

Fields:

id
createdAt
level (info/warn/error/critical)
service
eventType
message
metadata JSONB

Examples:

API failures
agent failures
workflow failures
Stripe failures
SendGrid failures
deployment events

──────────────────────────────
client_errors
──────────────────────────────

Fields:

id
createdAt
route
message
stack
userAgent
source
line
column

Connect to:

/api/client-errors

Store instead of console-only logging.

──────────────────────────────
query_failures
──────────────────────────────

Fields:

id
createdAt
route
queryKey
statusCode
message

Connect to:

QueryCache global error handler.

Return:

* schema
* migrations
* retention strategy

TASK 2 — RELIABILITY DASHBOARD
──────────────────────────────

Create:

/admin/reliability

Purpose:

Single place to monitor platform health.

Sections:

1. Platform Health
    * uptime
    * API success rate
    * query success rate
    * client crash rate
2. Error Trends
    * client errors
    * query failures
    * server errors
3. Agent Health
    * success rate
    * failures
    * dead-letter queue count
    * average execution time
4. Financial Systems
    * Stripe failures
    * wallet failures
    * billing errors
5. Communications
    * SendGrid failures
    * email success rate
6. Infrastructure
    * average API response time
    * slowest endpoints
    * request volume

Requirements:

* live refresh
* charts
* severity badges
* drill-down views

Return:

* page created
* widgets created
* data sources used

──────────────────────────────

1. Homepage
2. Login endpoint
3. Scheduling endpoint
4. Lead capture endpoint
5. Wallet endpoint
6. AI endpoint
7. Database connectivity

Record:

response time
status
error message

system_logs

Fields:

id
createdAt

details

Return:

* checks implemented
* schedule used
* alert conditions
