## 3. Save Without Sending

Current issue:

Approving sends immediately unless infrastructure policy prevents sending.

Add a true **Save & Review Later** action.

Behavior:

* Saves current subject/body edits.
* Keeps the proposal in the queue.
* Does not mark approved.
* Does not send.
* Does not trigger send guard.
* Does not execute the action.
* Updates the visible draft content so the admin sees saved edits next time.
* Optionally logs feedback if feedback chips or coaching text were provided.

Suggested route:

```ts
PATCH /api/ai-approvals/:id/save-edits
```

Payload:

```ts
{
  editedSubject?: string,
  editedBody?: string,
  feedbackTags?: string[],
  coachingFeedbackText?: string
}
```

Expected behavior:

* Persist edited subject/body.
* Create feedback/learning record if feedback provided.
* Keep `gmailAgentActions.status = "proposed"` or equivalent pending status.
* Return updated draft.

Acceptance criteria:

* User can edit a draft and save without sending.
* Saved edits persist after closing/reopening the drawer.
* Draft remains in approval queue.
* No email is sent.
* Feedback provided during save can still feed the learning loop.
