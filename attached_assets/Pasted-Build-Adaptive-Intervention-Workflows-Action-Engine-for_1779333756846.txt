Build Adaptive Intervention Workflows + Action Engine for TrainEfficiency.

GOAL:
Transform athlete intelligence into automated coaching workflows.

The system should:
- detect athlete issues
- generate interventions
- coordinate actions
- track outcomes
- adapt over time

====================================================
1. CORE CONCEPT
====================================================

Current:
The system identifies:
- low readiness
- missed workouts
- fatigue
- education noncompliance
- PR plateaus

Next:
The system should orchestrate adaptive responses.

Examples:

Low readiness
→ notify coach
→ suggest Recovery pathway
→ create hydration reminder
→ recommend lower workload
→ monitor for improvement

Missed workouts
→ send athlete reminder
→ notify coach after threshold
→ assign Training Habits module

PR plateau
→ recommend coach review
→ suggest program adjustment
→ suggest recovery intervention

====================================================
3. WORKFLOW ENGINE
====================================================

Allow rules-based workflows.

Example:

Workflow:
Low Readiness Recovery Workflow

Trigger:
3 low readiness scores in 7 days

Actions:
1. Create risk flag
2. Notify coach
3. Recommend Recovery pathway
4. Add hydration reminder
5. Create follow-up in 5 days
6. Monitor readiness trend

6. Athlete Disengagement
7. High Fatigue Deload Recommendation

====================================================
5. AI INTERVENTION ENGINE
====================================================

POST /api/org/adaptive-workflows/ai-recommend

AI receives:
- athlete scores
- readiness trends
- workout adherence
- education completion
- PR history
- coach notes
- active flags

AI returns:
- suggested intervention plan
- recommended education
- coaching suggestions
- monitoring recommendations
- follow-up timing

IMPORTANT:
AI never auto-applies program modifications.
