Build a dedicated Coach Athlete Detail Page for TrainEfficiency.

GOAL:
Move the coach athlete profile experience from a temporary dialog into a permanent coach-facing athlete page.

ROUTE:
Create:
/org/:slug/coach/athletes/:userId

AUTH:
Use existing org auth.
Only allow:
- coach
- owner

Security:
- Enforce org scope
- Coaches can only view athletes in teams they manage unless owner
- Never expose password/session data
- Athlete data must stay org-scoped

BACKEND:
Add endpoint:

GET /api/org/coach/athletes/:userId

Return:
- athlete profile
- org membership
- teams the athlete belongs to
- best PRs
- full PR history grouped by lift
- recent PR entries
- upcoming bookings
- past bookings
- coach notes
- available player card data

Add endpoint:

PATCH /api/org/coach/athletes/:userId/notes

Reuse existing coach_athlete_notes table if possible.

Create CoachAthleteDetailPage.
