Build a TrainEfficiency → Codex integration layer.

Goal:
Create a Software Improvement Agent that collects issues, bugs, failed jobs, workflow problems, and product opportunities from TrainEfficiency, then converts them into Codex-ready engineering tasks.

Important:
Do NOT allow Codex to directly modify production data, send emails, change Stripe, or execute business actions. This integration should create structured engineering tasks only.

Architecture:
CEO Heartbeat
→ Executive Agent
→ Revenue / Scheduling / Retention / Growth / Ops Agents
→ Software Improvement Agent
→ Codex Task Queue / GitHub Issue / PR workflow

Build the following:

1. Database tables

Create a table for software improvement tasks:

software_improvement_tasks

Statuses:

- rejected
- archived

2. Backend service

Create:

server/services/software-improvement-agent.ts

It should:
- scan workflow failures, dead letters, trigger audit logs, agent execution logs, attention inbox items, and user-reported bugs if those tables exist
- create or update software improvement tasks
- deduplicate similar issues
- calculate severity and priority
- generate a Codex-ready prompt

Codex prompt format:

You are working on TrainEfficiency.

Problem:
[problem_summary]

Business context:
[business_context]

Affected area:
[affected_area]

Suspected files/routes:
[suspected_files]

Reproduction steps:
[reproduction_steps]

Expected behavior:
[expected_behavior]

Constraints:
- Preserve multi-tenant isolation
- Do not break organization authentication
- Do not expose global data
- Do not modify production data
- Do not send emails, Stripe actions, or business actions
- Keep mobile responsive
- Add useful logging
- Add or update tests where possible

Acceptance checks:
- npm run check
- affected endpoint works
- no TypeScript errors
- no cross-org data leakage
- UI works on mobile if frontend is affected
