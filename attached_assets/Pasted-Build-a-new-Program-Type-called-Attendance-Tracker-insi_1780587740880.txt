Build a new Program Type called Attendance Tracker inside the Programs section.

Goal:
Allow organizations to create QR-code based attendance programs that track athlete participation, reward loyalty, generate leads, and automate communication.

=================================================
PROGRAM CREATION

Add a new program type:

* Scheduling Program
* PR Tracker
* Workout Builder
* Lead Capture Program
* Attendance Tracker ← NEW

Attendance Tracker should be configurable from the same Program Tools area.

=================================================
ATTENDANCE TRACKER BUILDER

Create a builder interface with the following sections:

Basic Settings

* Program Name
* Program Description
* Location
* Active / Inactive
* Start Date
* End Date

=================================================
ATHLETE INFORMATION FIELDS

Admins can configure which fields athletes must complete when checking in.

Default available fields:

* First Name
* Last Name
* Email
* Phone
* Sport
* Position
* School
* Graduation Year
* Team
* Age

For each field allow:

* Required
* Optional
* Hidden

Admin can add/remove fields.

Store configuration per attendance program.

=================================================
REWARDS BUILDER

Create a reward milestone system.

Examples:

5 Visits → EST Sticker
10 Visits → Water Bottle
15 Visits → Free T-Shirt
25 Visits → Free Session
50 Visits → Hoodie

Each reward tier should support:

* Visit Count Required
* Reward Name
* Reward Description
* Active / Inactive

Display athlete progress toward the next reward.

=================================================
QR CODE SYSTEM

Every attendance program automatically generates:

* Unique public URL
* Unique QR code

Example:

/attendance/blhs-speed

Athlete workflow:

Scan QR
→ Open attendance page
→ Complete fields
→ Submit attendance

No login required.

=================================================
ATTENDANCE SUBMISSION FLOW

When submitted:

1. Create attendance record
2. Update athlete attendance count
3. Calculate reward progress
4. Trigger confirmation email
5. Add/update athlete in lead database

If athlete already exists:

* Update attendance history
* Do not create duplicates

=================================================
LEAD DATABASE INTEGRATION

Every attendance submission should automatically create or update an athlete lead.

Use existing lead infrastructure.

Map:

Name
Email
Phone
Sport

Grad Year

into athlete leads database.

This allows agents and CRM workflows to work with attendees automatically.
