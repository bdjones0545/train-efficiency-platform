REPLIT PROMPT — REVENUE OUTCOME ENGINE (FULL CLOSED LOOP)

Objective:
Tie every AI action (manual or auto-executed) directly to real revenue outcomes and surface it clearly in the UI.

PHASE 1 — LINK ACTIONS TO OUTCOMES

Extend:

auto_execution_log
outreach_events
ai_revenue_events

Add:
- outcome_status:
  - pending
  - engaged
  - booked
  - won
  - lost
- outcome_value (numeric)
- outcome_source:
  - follow_up
  - initial_outreach
  - deal_pipeline
- outcome_timestamp

PHASE 2 — ATTRIBUTION LOGIC

When:
- booking is created
- deal is marked "won"

Match:
- prospect_id
- within attribution window (72h default)
- most recent action

Update:
- outcome_status = won
- outcome_value = booking/deal value

PHASE 3 — AUTO-EXECUTION FEEDBACK LOOP

When an auto-executed action leads to:
- reply → mark “engaged”
- meeting → mark “booked”
- deal → mark “won”

Store:
- time-to-outcome
- conversion chain

PHASE 4 — UI — “AI GENERATED REVENUE”

Add:

Cards:
- Today
- This Week
- This Month

Example:
“$1,240 generated by AI actions”

“From 6 actions — avg $206/action”
