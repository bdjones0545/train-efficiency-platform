Build Workout Execution + Readiness Layer for TrainEfficiency.

GOAL:
Create the feedback loop after Workout Builder Phase 1:

Generate → Assign → Execute → Monitor → Adapt

This should let athletes complete workouts with readiness/check-in data, and let coaches see when programs may need adjustment.

CONTEXT:
Workout Builder already exists and is powered by the TrainChat API.
Do NOT rebuild TrainChat.
Use TrainChat only for adaptive modification suggestions when needed.

====================================================
2. ATHLETE WORKOUT EXECUTION FLOW
====================================================

When athlete opens an assigned workout session:

Step 1:
Readiness Check-In

Ask:
- How ready do you feel today? 1–10
- Sleep quality 1–10
- Soreness 1–10
- Fatigue 1–10
- Stress 1–10
- Motivation 1–10
- Any pain areas?
- Notes

Step 2:
Workout Session View

Show:
- exercises
- sets/reps/load/time
- coach notes
- session focus

Step 3:
Exercise Logging

For each exercise:
- mark complete
- enter actual load/reps/time if applicable
- enter RPE
- notes

Step 4:
Finish Session

Save:
- completion log
- exercise logs
- readiness check-in

====================================================
4. ADAPTATION ENGINE
====================================================

Create a lightweight rules-first engine.

Rules examples:
- readinessScore <= 4 → suggest reduce intensity/volume
- sorenessLevel >= 8 → suggest recovery emphasis
- fatigueLevel >= 8 and RPE high → coach review
- missed 2+ sessions → reschedule or reduce density
- consistently easy RPE <= 5 → possible progression
- painAreas present → coach review required

The engine should create workout_adaptation_recommendations.

====================================================
5. TRAINCHAT API ADAPTATION SUPPORT
====================================================

Add optional coach action:

“Ask TrainChat for Modification”

When clicked:
- Send session/program context
- Send readiness data
- Send exercise logs
- Send coach notes
- Ask TrainChat for safe modification suggestions

Use existing:
trainChatClient.editProgram()
or trainChatClient.generateSession()

Do NOT automatically apply changes.

Coach must review:
- suggested changes
- rationale
- safety flags

Then choose:
- Accept
- Dismiss
- Edit manually
