# Goal

An athlete should not silently begin training when critical business requirements are incomplete.

This phase makes onboarding financially and legally ready without building a full billing or waiver platform.

The system should answer:

* Is this athlete financially ready?
* Has a waiver been completed?
* Is the athlete blocked from training?
* What should the coach/admin do next?

---

# Product Philosophy

Separate:

### Operational Readiness

Can this athlete begin training?

### Financial Readiness

Can this athlete be billed?

### Legal Readiness

Has required documentation been completed?

These become independent readiness dimensions.

# Scope

## 1. Billing Readiness Detection

Audit existing billing/payment schema.

Look for:

* Stripe customer IDs
* subscriptions
* payment methods
* invoices
* packages
* memberships
* athlete billing records

Create a reusable helper:

```ts
computeBillingReadiness(orgId, athleteUserId)
```

Return:

```ts
{
  ready: boolean,
  stripeCustomerExists: boolean,
  activeMembership: boolean,
  packageAssigned: boolean,
  paymentMethodOnFile: boolean,
  outstandingBalance: number,
  reason?: string
}
```

Use existing schema wherever possible.

Do not invent billing tables if they already exist.

## 2. Waiver Readiness

Audit the codebase.

Determine whether waivers already exist.

Possible locations:

* documents
* agreements
* signed forms
* uploads
* participant waivers

If supported:

Create:

```ts
computeWaiverReadiness()
```

Return:

```ts
{
    required: boolean,
    signed: boolean,
    signedAt?: Date,
    expiresAt?: Date,
    reason?: string
}
```

If waivers do not yet exist:

Do not build a complete waiver platform.

Instead:

* Add lightweight placeholder readiness.
* Return:

```ts
required: false
```

with a clear TODO.

---

## 3. Operational Readiness

Create one reusable helper:

```ts
computeOperationalReadiness()
```

Evaluate:

* Athlete account
* Program assigned
* First session scheduled
* Guardian requirements (if youth athlete)
* Billing
* Waiver
* PAIL initialized

Return:

```ts
{
    readyToTrain: boolean,
    blockingItems: [],
    warnings: [],
    readinessScore: number
}
```

Score:

0–100

Deterministic only.

No AI.
